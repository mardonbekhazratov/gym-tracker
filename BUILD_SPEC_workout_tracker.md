## 0. Product summary

A single-user, **offline-first** workout tracker for one fixed program
(3-day upper-body-focused split, Mon/Wed/Fri). Core job: log every set
(weight × reps × RIR), always show last session's numbers so the user can
beat them, track bodyweight, and surface the program's six rules.

- **No backend, no accounts, no network required.** All data lives on-device in IndexedDB.
- One React codebase → runs as a web app, installs as a PWA, wraps to a native Android APK via Capacitor.
- Units: kg default, lb toggle in settings.

## 3. Data model (Dexie tables / TypeScript types)

Use these exact field names. IDs are auto-increment numbers unless noted.

```ts
// db/db.ts
import Dexie, { Table } from 'dexie';

export interface Exercise {
  id?: number;
  slug: string;            // stable key, e.g. "incline-db-press"
  name: string;
  muscleGroup: MuscleGroup;
  defaultSets: number;
  repLow: number;
  repHigh: number;
  restSeconds: number;     // default rest, see §5 mapping
  alternatives: string[];  // free-text names
}

export type MuscleGroup =
  | 'chest' | 'back' | 'sideDelts' | 'rearDelts' | 'triceps'
  | 'biceps' | 'quads' | 'hamstrings' | 'calves' | 'abs' | 'shoulders';

export interface DayTemplate {
  id?: number;
  key: 'monday' | 'wednesday' | 'friday';
  label: string;                 // e.g. "Monday — Upper (Push focus)"
  exerciseSlugs: string[];       // ordered
}

export interface Session {
  id?: number;
  date: string;                  // ISO yyyy-mm-dd
  dayKey: 'monday' | 'wednesday' | 'friday';
  notes?: string;
  completed: boolean;
}

export interface SetLog {
  id?: number;
  sessionId: number;
  exerciseSlug: string;          // store slug, not numeric id (survives reseed)
  setNumber: number;             // 1-based
  weightKg: number;              // always store in kg internally
  reps: number;
  rir: number;                   // reps in reserve, 0–5
  timestamp: number;             // Date.now()
}

export interface BodyWeight {
  id?: number;
  date: string;                  // ISO yyyy-mm-dd
  weightKg: number;
}

export interface Settings {
  id?: number;                   // single row, id = 1
  units: 'kg' | 'lb';
  programStartDate: string;      // ISO; used for week count + deload
  lastDeloadDate?: string;       // ISO
  proteinPerKg: number;          // default 1.8
  bodyweightForProteinKg?: number;
}

export class AppDB extends Dexie {
  exercises!: Table<Exercise, number>;
  dayTemplates!: Table<DayTemplate, number>;
  sessions!: Table<Session, number>;
  setLogs!: Table<SetLog, number>;
  bodyWeights!: Table<BodyWeight, number>;
  settings!: Table<Settings, number>;

  constructor() {
    super('workoutTrackerDB');
    this.version(1).stores({
      exercises:    '++id, slug, muscleGroup',
      dayTemplates: '++id, key',
      sessions:     '++id, date, dayKey',
      setLogs:      '++id, sessionId, exerciseSlug',
      bodyWeights:  '++id, date',
      settings:     '++id',
    });
  }
}
export const db = new AppDB();
```

**Key query — "beat last session":** for a given `exerciseSlug`, find the most
recent `session.date` strictly before today that has setLogs for that slug,
and return those setLogs ordered by `setNumber`. Render them as greyed "ghost"
placeholders in each SetRow.

> different rep/rest targets, hence distinct slugs (`-f` suffix for Friday).

### 5.2 Day templates

- `monday`    → [incline-db-press, flat-bench-press, lat-pulldown, chest-supported-row, db-lateral-raise, triceps-pushdown]
- `wednesday` → [back-squat, romanian-deadlift, leg-press, seated-leg-curl, standing-calf-raise, hanging-leg-raise]
- `friday`    → [weighted-pull-ups, overhead-press, barbell-row, incline-db-press-f, dumbbell-curl, db-lateral-raise-f, rear-delt-flye]

### 5.3 Reference content (show in app where relevant, not editable)

**Pre-workout (10–12 min):**
- General warm-up 5 min: 3–5 min easy cardio (bike/row/jog), light sweat.
- Dynamic stretch 3–4 min: arm circles 10 each way; shoulder dislocations ×10; cat-cow ×10; world's greatest stretch 5/side; bodyweight squats ×10; leg swings 10 each leg (front-back + side-side).
- Specific warm-up before first compound: empty bar ×10 → ~40% ×5 → ~60% ×3 → ~80% ×1–2 → working sets. Follow-up exercises same muscle: one light set.

**Weekly volume targets (for §6 dashboard):**
- chest 9 /wk · back 9 /wk · sideDelts 7 /wk · triceps 3 direct · biceps 3 direct · quads 8–11 · hamstrings 8–11. Evidence-based range 10–20 sets/wk/muscle.

**Post-workout:** static stretch 5 min (hold 20–30 s, trained muscles); protein 30–40 g within ~2 h.

**The Six Critical Rules (display on a Rules card + drive reminders):**
1. Log every set; beat last session by +1 rep or +2.5 kg (progressive overload).
2. Train at 0–2 RIR; only the last set of an exercise nears true failure.
3. Protein 1.6–2.2 g/kg bodyweight daily (70 kg → 110–150 g). Non-negotiable.
4. Sleep 7–9 h.
5. Deload every 6–8 weeks: drop to ~60% load for one week.
6. Run the program ≥12 weeks before changing anything.

## 6. Features → implementation notes

| Feature | Notes |
|---|---|
| **Set logger** | SetRow per set: weight, reps, RIR. Pre-fill target reps from repLow–repHigh. Numeric keypad inputs. Tap "+ set" to add beyond default. |
| **Last-session ghosts** | Use the query in §3; show prior weight×reps as placeholder text in each SetRow. |
| **Rest timer** | On set save, start countdown from exercise.restSeconds. Show remaining; allow skip/+30s. Phase 5: local notification + haptic on finish. |
| **Exercise swap** | Per exercise, open sheet listing `alternatives`; selected swap applies to current session only (don't mutate template). Log under original slug or a `swappedTo` note. |
| **Bodyweight** | Quick add (date + kg); line chart over time. Feeds protein target. |
| **Per-exercise progress** | Chart top-set weight and/or Epley est-1RM over time (`lib/epley.ts`). |
| **Weekly volume dashboard** | Count working sets per muscleGroup in current ISO week; compare to §5.3 targets and the 10–20 band; color code. |
| **Deload banner** | If weeks-since(programStartDate or lastDeloadDate) ≥ 6, show banner (Rule 5). Button: "Mark deload done" → sets lastDeloadDate. |
| **Protein badge** | grams = round(bodyweightForProteinKg × proteinPerKg). Show daily target. |
| **12-week progress** | Show current week N/12 from programStartDate. |

## 7. Screens

- **Today** (default route `/`): auto-select today's dayKey (Mon→monday, Wed→wednesday, Fri→friday; other days → let user pick). List exercises with target sets×reps; tap to expand SetRows w/ ghosts; rest timer after each save. DeloadBanner + ProteinBadge at top when relevant. "Finish session" marks completed.
- **History** (`/history`): list/calendar of sessions newest first; tap to view/edit logs + notes.
- **Progress** (`/progress`): exercise picker → load/1RM chart; bodyweight chart; weekly volume dashboard.
- **Settings** (`/settings`): kg/lb toggle; programStartDate; proteinPerKg + bodyweight; export/import JSON; reset data (confirm).

UX: dark mode default, high contrast, large tap targets, thumb-reachable bottom nav (Today / History / Progress / Settings), portrait-locked. Tune for S24 Ultra tall display.

## 10. Capacitor / Android (Phase 4) — commands

```bash
# from project root, after the web app builds
npm i @capacitor/core @capacitor/cli
npx cap init "Workout Tracker" com.ismoil.workouttracker --web-dir=dist
npm i @capacitor/android
npx cap add android

npm run build          # produce dist/
npx cap sync           # copy web build into android project (re-run after each change)
npx cap open android   # opens Android Studio → Run/Build APK
```

**Onto the S24 Ultra:** enable Developer Options (Settings → About phone → tap
Build number 7×), enable USB debugging, connect USB, Run from Android Studio.
Or build `app-debug.apk`, copy to phone, allow "install unknown apps," open it.
For a keepable build: generate a signing key → signed `.apk`/`.aab` (optionally
Play Console internal track). All buildable on Windows/Linux.
